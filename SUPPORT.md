# Support

## Where To Get Help

- Bugs and reproducible defects: open a GitHub issue using the bug report template.
- Product, workflow, or platform improvements: open a GitHub issue using the change request template.
- Security concerns: follow [SECURITY.md](/home/ntr/Documents/dev/maasai-lang/SECURITY.md) and report privately.
- Sensitive operational issues involving secrets, access, or private infrastructure: use an approved internal NorthernTribe-Research channel rather than a public issue.

## What To Include

For faster triage, include:

- the affected area such as Space, training, dataset, CI/CD, Blacksmith, Kaggle, or Hugging Face
- exact commands or workflow runs
- logs or stack traces with secrets removed
- the branch, commit, and environment involved

## Support Boundaries

- Public issues are appropriate for non-sensitive repository problems and feature requests.
- Private operational incidents, credentials, and vulnerability reports should not be posted publicly.
- Best-effort community help may be available, but organization-maintained operational work is prioritized for NorthernTribe-Research maintainers and approved collaborators.
